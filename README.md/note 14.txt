# Notebook 14 – OSOPSS Day-Ahead Scheduling with ASAP EV Load

## Objective

The objective of this notebook is to extend the OSOPSS day-ahead PV+BESS scheduling procedure by including EV charging demand as a fixed, non-controllable load.

This follows the supervisor's recommendation:

* first use the optimally sized PV+BESS system obtained from the OSOPSS joint sizing procedure,
* then include the EV ASAP charging profile as additional load,
* then compute the best PV+BESS operation using the day-ahead scheduling approach from the course.

In this notebook, EV charging is not yet optimized. It is treated as a fixed load profile.

---

## Input System

The optimally sized PV+BESS system obtained in Notebook 13 is used.

System parameters:

* PV size: 588.77 kWp
* Battery energy capacity: 403.43 kWh
* Battery converter size: 103.25 kW

Battery assumptions:

* Charging efficiency: 90%
* Discharging efficiency: 90%
* Maximum depth of discharge: 85%

---

## Representative Scenarios

Three representative EV demand days are considered:

| Scenario          | Day |
| ----------------- | --: |
| Low-demand day    | 106 |
| Medium-demand day |  20 |
| High-demand day   |  95 |

These days represent different combinations of EV charging demand and PV availability.

---

## Method

For each representative day, two cases are compared:

1. Building load only
2. Building load + ASAP EV charging profile

The ASAP EV charging profile is added directly to the building demand. Therefore, the total load becomes:

Total load = Building load + ASAP EV charging load

The PV+BESS operation is then optimized using the OSOPSS day-ahead scheduling structure.

The optimization includes:

* PV production
* Battery charging
* Battery discharging
* Battery state of charge
* Grid import
* Grid export
* Peak grid import

The objective includes electricity import/export cost and peak grid power cost.

---

## Low-Demand Scenario

For the low-demand day, the EV charging demand is only 37 kWh.

Results:

* Cost increase: 2.61 EUR
* Grid import increase: 0 kWh
* Peak increase: 0 kW
* Grid export reduction: 37 kWh

Interpretation:

The EV demand is small and is fully absorbed by local PV+BESS operation. It does not increase grid import or peak demand. Instead, the EV load reduces exported PV energy by approximately the same amount as the EV demand.

---

## Medium-Demand Scenario

For the medium-demand day, the EV charging demand is 303 kWh.

Results:

* Cost increase: 42.63 EUR
* Grid import increase: 308.11 kWh
* Peak increase: 12.01 kW
* Grid export change: 0 kWh

Interpretation:

This is the most critical scenario. PV availability is limited on this day, so adding ASAP EV charging increases both grid import and peak grid demand. This leads to a clear increase in the daily operating cost.

This scenario shows that when PV production is insufficient, non-controllable EV charging can increase both energy purchases and peak demand.

---

## High-Demand Scenario

For the high-demand day, the EV charging demand is 509 kWh.

Results:

* Cost increase: 22.13 EUR
* Grid import increase: 0 kWh
* Peak increase: 0 kW
* Grid export reduction: 509 kWh

Interpretation:

Although EV demand is high, PV production is also high on this day. The EV load is supplied locally by the PV+BESS system and does not increase grid import or peak demand.

The main effect is that exported PV energy is reduced by approximately 509 kWh, which corresponds to the EV demand.

This means that, for this day, every EV kWh is supplied locally instead of being exported to the grid.

---

## Main Conclusion

The impact of ASAP EV charging depends strongly on the correlation between EV demand and PV availability.

When PV production is abundant, EV charging mainly reduces exported PV energy and does not increase grid import.

When PV production is limited, EV charging increases grid import, peak grid demand, and daily operating cost.

This confirms the need for the next stage of the project: smart charging optimization, where EV charging is no longer treated as a fixed load but becomes a controllable decision variable in the day-ahead scheduling problem.
